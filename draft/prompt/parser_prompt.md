Я разрабатываю Django-приложение `intellectual_property` для управления объектами интеллектуальной собственности (РИД). У меня есть каталоги открытых данных ФИПС Роспатента в формате CSV для разных типов РИД: изобретения, полезные модели, промышленные образцы, программы для ЭВМ, базы данных, топологии микросхем.

**Мои модели уже созданы:**
- `IPObject` — основная модель для всех РИД
- `Person` — физические лица (авторы, правообладатели)
- `Organization` — организации-правообладатели
- `FOIV`, `RFRepresentative` — для госорганов
- `Country`, `City`, `Region` — география
- и другие вспомогательные модели

**Задача:**  
Нужно написать management command `pars_fips_catalogue.py`, которая будет:
1. Принимать CSV файл каталога
2. Парсить данные в зависимости от типа РИД
3. Сохранять в БД с правильным связыванием сущностей
4. Избегать дубликатов организаций и физлиц
5. Правильно форматировать текст (регистр букв, кавычки, пробелы)

**Технические требования:**
- Использовать `pandas` для чтения CSV
- Обрабатывать все типы РИД одним универсальным механизмом
- Кэшировать уже найденные сущности (Person, Organization)
- Предусмотреть пакетную обработку (batch operations)
- Режимы `--dry-run`, `--only-active`, фильтр по году
- Поддержка разных кодировок CSV

**Проблемы, которые нужно решить:**
- Организации могут называться по-разному (ФГУП "НИИ" и Федеральное государственное унитарное предприятие "Научно-исследовательский институт")
- В кавычках слова должны быть с большой буквы
- Римские цифры (VIII, IV) нужно сохранять в верхнем регистре
- Предлоги и союзы должны быть с маленькой буквы
- Первое слово названия РИД — с большой, остальные с маленькой
